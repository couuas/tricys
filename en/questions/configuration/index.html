    2. Configuration & Execution

    ⚙️ Configuration & Execution¶

    Q: How do I run a simulation?¶

    TRICYS provides several ways to run:

    1. Command-Line Interface (CLI): 

    # Use default config file config.json
tricys

# Specify a config file
tricys -c my_config.json

# Run an analysis task
tricys analysis -c analysis_config.json

    2. Graphical User Interface (GUI): 

    tricys gui

    3. Interactive Examples: 

    # Run interactive menu for all examples
tricys example

# Run basic simulation examples
tricys basic example

# Run analysis examples
tricys analysis example

    Q: How do I understand the output files?¶

    After the simulation is complete, results are saved in a timestamped subdirectory within the results/ directory:

    Filename Description
    simulation_result.csv Detailed results of a single simulation, containing all variables over time.
    sweep_results.csv Aggregated results of a parameter sweep, where each column represents one run.
    tricys.log Detailed run log, containing debug information.
    config.json A backup of the full configuration used for this run.
    *_report.html Auto-generated analysis report (for analysis tasks only).
    *.png / *.csv Various charts and data exports.

    CSV File Structure:

    time,sds.I[1],blanket.I[1],...
0.0,10.5,2.3,...
0.5,10.4,2.35,...
...
    • First column: time (time axis)
    • Other columns: model variables, column names correspond to variables in variableFilter.

    Sweep Result File (sweep_results.csv): 

    time,blanket.TBR=1.05_sds.I[1],blanket.TBR=1.1_sds.I[1],...

    Column name format: <param_name>=<param_value>_<var_name>

    Q: How do I define complex parameter sweeps?¶

    TRICYS supports various parameter sweep formats:

    Feature Format Example Description
    Discrete List [v1, v2, ...] [6, 12, 18] A set of discrete values
    Arithmetic Sequence (Range) "start:stop:step" "1.05:1.15:0.05" Start, stop, and step size
    Linear Spacing (Linspace) "linspace:start:stop:num" "linspace:10:20:5" Generates num evenly spaced points
    Logarithmic Spacing (Logspace) "log:start:stop:num" "log:1:1000:4" Generates num logarithmically spaced points
    Read from File "file:path:column" "file:data.csv:voltage" Reads from a specified column in a CSV file

    Example Configuration: 

    {
    "simulation_parameters": {
        "blanket.TBR": [1.05, 1.1, 1.15, 1.2],
        "plasma.fb": "linspace:0.01:0.1:10",
        "tep_fep.to_SDS_Fraction[1]": "log:0.1:1.0:5"
    }
}

    Q: How do I filter output variables?¶

    Use the variableFilter parameter (regular expression):

    {
    "simulation": {
        "variableFilter": "time|sds.I[1]|blanket.*|div.I\\[.*\\]"
    }
}

    Common Patterns: - time: time variable (must be included) - sds.I[1]: exact match - blanket.*: match all variables starting with blanket - .*\.I\[.*\]: match the I array for all subsystems

    Note

    In JSON, backslashes need to be escaped: \[ is written as \\[.

    Q: The simulation takes a long time, how can I speed it up?¶

    You can take the following optimization measures:

    1. Enable Concurrent Execution: 

    {
    "simulation": {
        "concurrent": true,
        "max_workers": 4
    }
}

    2. Reduce Output Variables: 

    {
    "simulation": {
        "variableFilter": "time|sds.I[1]"  // Save only key variables
    }
}

    3. Increase the Time Step (trade-off with accuracy): 

    {
    "simulation": {
        "step_size": 1.0  // Increase from 0.5 to 1.0
    }
}

    4. Reduce the Number of Sweep Points: 

    {
    "simulation_parameters": {
        "blanket.TBR": "linspace:1.05:1.15:5"  // Reduce from 20 to 5
    }
}
