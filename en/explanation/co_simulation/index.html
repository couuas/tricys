    3. Co-simulation Principle

    CSV 数据替换方法对比¶

    在协同仿真中，TRICYS 提供了两种将外部数据注入 Modelica 模型的方法。

    方法一：拦截器模式（Interceptor Pattern）¶

    工作原理¶

    在原模型和下游系统之间插入一个拦截器模型，拦截器接收两个输入源： 1. 原模型的物理输出 2. CSV 文件的数据

    通过配置参数动态选择使用哪个数据源。

    架构图¶

    原系统结构：
  [上游模型] --> [子模型] --> [下游模型]

拦截器模式结构：
  [上游模型] --> [子模型] --> [拦截器] --> [下游模型]
                               ↑
                           [CSV 数据]

    实现步骤¶

    1. 生成拦截器模型代码（包含 CombiTimeTable 和条件方程）
    2. 将拦截器模型添加到包中
    3. 在系统模型中实例化拦截器
    4. 重定向连接：
    5. connect(upstream, submodel.port) → connect(upstream, submodel.port)（保持不变）
    6. connect(submodel.port, downstream) → connect(interceptor.final_port, downstream)
    7. 新增：connect(submodel.port, interceptor.physical_port)

    拦截器模型示例¶

    within example_model;

model Plasma_Interceptor
  // 接收原模型输出
  Modelica.Blocks.Interfaces.RealInput physical_to_Pump[5];

  // 最终输出
  Modelica.Blocks.Interfaces.RealOutput final_to_Pump[5];

  protected
    parameter String fileName = "plasma_data.csv";

    // 列映射参数：[time, y1, y2, y3, y4, y5]
    // 如果某列设为 1，表示使用物理数据而非 CSV
    parameter Integer columns_to_Pump[6] = {1, 2, 3, 4, 5, 6};

    // CSV 读取器
    Modelica.Blocks.Sources.CombiTimeTable table_to_Pump(
      tableName="csv_data_to_Pump",
      fileName=fileName,
      columns=columns_to_Pump,
      tableOnFile=true
    );

  equation
    // 逐元素选择数据源
    for i in 1:5 loop
      final_to_Pump[i] = if columns_to_Pump[i+1] <> 1 
                         then table_to_Pump.y[i]   // 使用 CSV
                         else physical_to_Pump[i]; // 使用物理数据
    end for;

end Plasma_Interceptor;

    优点¶

    • ✅ 非侵入性：不修改原模型，保持原模型完整性
    • ✅ 灵活性高：可通过参数动态切换物理/CSV 模式
    • ✅ 细粒度控制：向量端口的每个元素可独立配置
    • ✅ 可逆性强：可以轻松移除拦截器恢复原系统
    • ✅ 便于调试：可以同时观察物理输出和 CSV 数据

    缺点¶

    • ❌ 结构复杂：需要创建额外的模型文件
    • ❌ 修改系统拓扑：需要重定向系统模型中的连接
    • ❌ 代码生成复杂：需要复杂的正则表达式处理连接语句
    • ❌ 增加组件数：系统中增加了额外的组件实例

    方法二：直接替换模式（Direct Replacement Pattern）¶

    工作原理¶

    直接修改子模型的内部实现： 1. 备份原模型文件 2. 保留输入输出接口声明 3. 删除所有内部方程和变量 4. 添加 CombiTimeTable 组件 5. 创建简单映射：output = table.y

    架构图¶

    原系统结构：
  [上游模型] --> [子模型] --> [下游模型]

直接替换模式：
  [上游模型] --> [子模型(CSV版)] --> [下游模型]
                     ↑
                 [CSV 数据]

    实现步骤¶

    1. 备份原模型文件（例如：Plasma.mo → Plasma_backup.mo）
    2. 读取原模型，解析输入输出端口
    3. 生成新的模型代码：
    4. 保留端口声明
    5. 删除所有内部变量、参数、方程
    6. 添加 CombiTimeTable
    7. 创建映射方程
    8. 覆盖原文件

    替换后模型示例¶

    within example_model;

model Plasma
  // 原有端口声明保持不变
  Modelica.Blocks.Interfaces.RealInput pulseInput;
  Modelica.Blocks.Interfaces.RealOutput from_Fueling_System[5];
  Modelica.Blocks.Interfaces.RealOutput to_FW[5];
  Modelica.Blocks.Interfaces.RealOutput to_Div[5];
  Modelica.Blocks.Interfaces.RealOutput to_Pump[5];

protected
  parameter String fileName = "plasma_data.csv";

  // CSV 数据源
  Modelica.Blocks.Sources.CombiTimeTable table_to_Pump(
    tableName="csv_data_to_Pump",
    fileName=fileName,
    columns={1, 2, 3, 4, 5, 6},  // time + 5 data columns
    tableOnFile=true
  );

  Modelica.Blocks.Sources.CombiTimeTable table_to_FW(
    tableName="csv_data_to_FW",
    fileName=fileName,
    columns={1, 7, 8, 9, 10, 11},
    tableOnFile=true
  );

  // ... 其他端口的 table 定义 ...

equation
  // 直接映射 CSV 到输出（忽略输入）
  for i in 1:5 loop
    to_Pump[i] = table_to_Pump.y[i];
    to_FW[i] = table_to_FW.y[i];
    to_Div[i] = table_to_Div.y[i];
    from_Fueling_System[i] = table_from_Fueling_System.y[i];
  end for;

end Plasma;

    优点¶

    • ✅ 结构简单：不增加额外组件，系统拓扑不变
    • ✅ 实现简洁：不需要修改连接关系
    • ✅ 代码清晰：生成的模型代码更易读
    • ✅ 性能优越：减少了中间连接，可能更高效
    • ✅ 兼容性好：对系统模型无侵入

    缺点¶

    • ❌ 修改原文件：直接覆盖原模型（虽然有备份）
    • ❌ 灵活性低：无法动态切换物理/CSV 模式
    • ❌ 输入被忽略：输入端口虽然保留但不再使用，可能产生警告
    • ❌ 调试困难：无法同时观察物理输出和 CSV 数据
    • ❌ 恢复麻烦：需要手动恢复备份文件

    使用场景建议¶

    使用拦截器模式（方法一）当：¶

    1. 需要在物理模型和 CSV 数据之间动态切换
    2. 需要精细控制哪些输出使用 CSV，哪些使用物理计算
    3. 希望保持原模型完整性，便于版本管理
    4. 需要进行对比验证（物理 vs CSV）
    5. 在开发和调试阶段

    使用直接替换模式（方法二）当：¶

    1. 确定要完全使用 CSV 数据替换模型
    2. 原模型不再需要物理计算
    3. 希望简化系统结构
    4. 在生产环境或批量仿真中
    5. 模型性能是关键考虑因素

    代码使用示例¶

    方法一：拦截器模式¶

    from tricys.core.interceptor import integrate_interceptor_model

interception_configs = [
    {
        "submodel_name": "example_model.Plasma",
        "instance_name": "plasma",
        "output_ports": [
            {"name": "to_Pump", "dim": 5, "default_column": [1, 2, 3, 4, 5, 6]},
        ],
        "csv_file": "plasma_data.csv",
    }
]

result = integrate_interceptor_model(
    package_path="example_model/package.mo",
    model_name="example_model.Cycle",
    interception_configs=interception_configs,
)

    方法二：直接替换模式¶

    from tricys.core.interceptor import replace_submodels_with_csv

replacement_configs = [
    {
        "submodel_name": "example_model.Plasma",
        "output_ports": [
            {"name": "to_Pump", "dim": 5, "default_column": [1, 2, 3, 4, 5, 6]},
            {"name": "to_FW", "dim": 5, "default_column": [1, 7, 8, 9, 10, 11]},
            {"name": "to_Div", "dim": 5, "default_column": [1, 12, 13, 14, 15, 16]},
        ],
        "csv_file": "plasma_data.csv",
    }
]

result = replace_submodels_with_csv(
    package_path="example_model/",
    replacement_configs=replacement_configs,
)

# 恢复原模型
import shutil
backup_path = result['replaced_models'][0]['backup_path']
original_path = result['replaced_models'][0]['original_path']
shutil.copy2(backup_path, original_path)
