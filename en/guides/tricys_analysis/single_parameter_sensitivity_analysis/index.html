    单参数敏感性分析¶

    单参数敏感性分析是 tricys 的核心功能之一，旨在研究单个独立参数的变化如何影响一系列用户定义的关键性能指标 (KPIs)。

    该功能会自动运行一系列仿真（每次仿真对应独立参数的一个取值），计算每个仿真结果的性能指标，并生成图表来直观地展示它们之间的关系。

    运行命令¶

    敏感性分析任务由 tricys analysis 命令执行：

    tricys analysis -c single_parameter_sensitivity_analysis.json

    配置文件示例¶

    敏感性分析的配置定义在 sensitivity_analysis 字段中，这是一个结构较为复杂的对象。

    {
    "paths": { ... },
    "simulation": { ... },
    "sensitivity_analysis": {
        "enabled": true,
        "analysis_cases": [
            {
                "name": "TBR_Analysis",
                "independent_variable": "blanket.TBR",
                "independent_variable_sampling": [1.05, 1.1, 1.15, 1.2],
                "dependent_variables": [
                    "Startup_Inventory",
                    "Self_Sufficiency_Time",
                    "Doubling_Time"
                ],
                "plot_type":"line",
                "combine_plots":true,
                "sweep_time":["sds.I[1]"]
            }
        ],
        "metrics_definition": {
            "Startup_Inventory": {
                "source_column": "sds.I[1]",
                "method": "calculate_startup_inventory"
            },
            "Self_Sufficiency_Time": {
                "source_column": "sds.I[1]",
                "method": "time_of_turning_point"
            },
            "Doubling_Time": {
                "source_column": "sds.I[1]",
                "method": "calculate_doubling_time"
            }
        },
        "glossary_path": "../../example_glossary/example_glossary.csv",
        "unit_map": {
            "Doubling_Time": {"unit": "days", "conversion_factor": 24},
            "Startup_Inventory": {"unit": "kg", "conversion_factor": 1000}
        }
    }
}

    配置项详解¶

    1. metrics_definition (指标定义)¶

    这是最关键的部分，用于定义您关心的性能指标 (KPIs)，也就是图表中的因变量。

    • 结构: 一个字典，其中每个键都是您为指标赋予的唯一名称（如 Startup_Inventory）。
    • 值: 一个描述如何计算该指标的对象。
      • source_column: 用于计算的原始数据来源，即仿真结果 (.csv) 中的列名。
      • method: tricys.analysis.metric 模块中用于计算的函数名。tricys 内置了多种常用指标计算函数，同时也支持用户自定义。

    2. analysis_cases (分析案例)¶

    这是一个列表，允许您在单个配置文件中定义多个独立的敏感性分析任务。

    • name: 为该分析案例指定的唯一名称，将用于命名输出文件。
    • independent_variable: 要进行扫描的独立参数的完整模型路径。
    • independent_variable_sampling: 为独立参数提供的一组扫描值（支持离散列表和高级范围格式）。
    • dependent_variables: 一个列表，包含要分析的因变量（即在 metrics_definition 中定义的指标名称）。
    • plot_type: 生成的敏感性图表的类型，如 "line" (线图)。
    • combine_plots: 是否将多个因变量绘制在同一张图表中。
    • sweep_time: 一个包含原始变量名的列表。对于此列表中的每个变量，程序会生成一张“族曲线”图，即将每次参数扫描得到的时间演化曲线绘制在同一张图上，便于比较动态行为的差异。

    3. glossary_path 和 unit_map (报告美化)¶

    • glossary_path: 与基准工况分析中的作用相同，用于将变量名翻译成易于理解的术语。
    • unit_map: 一个字典，用于自定义报告图表中的单位。
      • 键: 变量名或指标名。
      • 值: 包含 unit (单位字符串) 和 conversion_factor (从原始单位到目标单位的换算系数) 的对象。例如，如果仿真时间单位是小时，通过 "conversion_factor": 24 可以将 Doubling_Time 的单位换算为天。

    分析报告输出¶

    运行结束后，tricys-ana 会为 analysis_cases 中的每一个案例生成一份独立的 HTML 报告，例如 TBR_Analysis_sensitivity_report.html。

    每份报告包含： 1. 敏感性分析图: - 展示因变量（如 Startup_Inventory）如何随独立变量（如 blanket.TBR）的变化而变化。 2. 族谱图 (Family of Curves): - 根据 sweep_time 的设置，将被扫描变量（如 sds.I[1]）在不同独立参数取值下的时间演化曲线绘制在一起。 3. 原始数据表: - 一个表格，清晰地列出每次仿真运行的独立参数值和计算出的所有因变量指标值。
