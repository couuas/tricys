    Single-Parameter Sensitivity Analysis¶

    Single-parameter sensitivity analysis is one of the core features of tricys, designed to study how changes in a single independent parameter affect a series of user-defined Key Performance Indicators (KPIs).

    This feature automatically runs a series of simulations (one for each value of the independent parameter), calculates performance metrics for each simulation result, and generates plots to visually represent the relationships between them.

    1. Command¶

    The sensitivity analysis task is executed by the tricys analysis command, or tricys can automatically detect and run the analysis:

    tricys analysis -c single_parameter_sensitivity_analysis.json

or

tricys -c single_parameter_sensitivity_analysis.json

    2. Configuration File Example¶

    The sensitivity analysis configuration is defined in the sensitivity_analysis field, which is a relatively complex object.

    {
    "paths": { ... },
    "simulation": { ... },
    "sensitivity_analysis": {
        "enabled": true,
        "analysis_cases": [
            {
                "name": "TBR_Analysis",
                "independent_variable": "blanket.TBR",
                "independent_variable_sampling": [1.05, 1.1, 1.15, 1.2],
                "dependent_variables": [
                    "Startup_Inventory",
                    "Self_Sufficiency_Time",
                    "Doubling_Time"
                ],
                "plot_type":"line",
                "combine_plots":true,
                "sweep_time":["sds.I[1]"]
            }
        ],
        "metrics_definition": {
            "Startup_Inventory": {
                "source_column": "sds.I[1]",
                "method": "calculate_startup_inventory"
            },
            "Self_Sufficiency_Time": {
                "source_column": "sds.I[1]",
                "method": "time_of_turning_point"
            },
            "Doubling_Time": {
                "source_column": "sds.I[1]",
                "method": "calculate_doubling_time"
            }
        },
        "glossary_path": "../../example_glossary/example_glossary.csv",
        "unit_map": {
            "Doubling_Time": {"unit": "days", "conversion_factor": 24},
            "Startup_Inventory": {"unit": "kg", "conversion_factor": 1000}
        }
    }
}

    3. Configuration Details¶

    3.1. metrics_definition¶

    This is the most critical part, used to define the performance indicators (KPIs) you care about, which are the dependent variables in the plots.

    • Structure: A dictionary where each key is a unique name you assign to the metric (e.g., Startup_Inventory).
    • Value: An object describing how to calculate the metric.
      • source_column: The source of the raw data for the calculation, i.e., the column name in the simulation result (.csv).
      • method: The name of the function in the tricys.analysis.metric module used for the calculation. tricys has several built-in functions for common metrics and also supports user-defined ones.

    3.2. analysis_cases¶

    This is a list that allows you to define multiple independent sensitivity analysis tasks in a single configuration file.

    • name: A unique name for this analysis case, which will be used for naming output files.
    • independent_variable: The full model path of the independent parameter to be swept.
    • independent_variable_sampling: A set of sweep values for the independent parameter (supports discrete lists and advanced range formats).
    • dependent_variables: A list of dependent variables to be analyzed (i.e., the metric names defined in metrics_definition).
    • plot_type: The type of sensitivity plot to generate, e.g., "line".
    • combine_plots: Whether to plot multiple dependent variables on the same chart.
    • sweep_time: A list of raw variable names. For each variable in this list, the program will generate a "family of curves" plot, drawing the time evolution curve from each parameter sweep run on the same chart for easy comparison of dynamic behavior.

    3.3. glossary_path and unit_map (Report Enhancement)¶

    • glossary_path: Same function as in the baseline condition analysis, used to translate variable names into understandable terms.
    • unit_map: A dictionary used to customize the units in the report charts.
      • Key: The variable or metric name.
      • Value: An object containing unit (a string for the unit) and conversion_factor (the conversion factor from the original unit to the target unit). For example, if the simulation time unit is hours, a "conversion_factor": 24 can convert the Doubling_Time unit to days.

    4. Analysis Report Output¶

    After running, tricys-ana will generate a separate HTML report for each case in analysis_cases, for example, TBR_Analysis_sensitivity_report.html.

    Each report contains: 1. Sensitivity Analysis Plot: - Shows how dependent variables (e.g., Startup_Inventory) change with the independent variable (e.g., blanket.TBR). 2. Family of Curves Plot: - Based on the sweep_time setting, plots the time evolution curves of a swept variable (e.g., sds.I[1]) for different values of the independent parameter together. 3. Raw Data Table: - A table that clearly lists the independent parameter value and the calculated values of all dependent variable metrics for each simulation run.
