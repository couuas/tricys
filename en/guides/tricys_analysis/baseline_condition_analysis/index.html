    Baseline Condition Analysis¶

    Baseline Condition Analysis is a core feature of the tricys analysis toolset. It is used to comprehensively evaluate the system behavior under a single, defined parameter configuration (i.e., the "baseline condition") and automatically generate a standardized analysis report.

    The baseline condition analysis program is essentially a post-processor for the simulation data of the current 0D tritium fuel cycle system model. Therefore, tricys extends the basic analysis, plotting, and reporting functions by integrating with the built-in base_analysis.py post-processing function. This is very useful for quickly understanding the model's dynamic characteristics and tritium inventory distribution at a specific design point.

    1. Command¶

    tricys -c baseline_condition_analysis.json

    2. Configuration File Example¶

    This analysis configuration is essentially a single simulation run, followed by a specific post-processing step. Therefore, the configuration file does not include a simulation_parameters sweep section.

    {
    "paths": {
        "package_path": "../../example_model_single/example_model.mo"
    },
    "simulation": {
        "model_name": "example_model.Cycle",
        "variableFilter": "time|sds.I[1]|div.I[1]|cps.I[1]|tes.I[1]|blanket.I[1]|i_iss.I[1]|wds.I[1]|o_iss.I[1]",
        "stop_time": 2000.0,
        "step_size": 0.5
    },
    "post_processing": [
        {
           "module": "tricys.postprocess.baseline_analysis",
           "function": "baseline_analysis",
           "params": {
                "detailed_var": "sds.I[1]",
                "glossary_path": "../../example_glossary/example_glossary.csv"
            }
        }
    ]
}

    3. Configuration Details¶

    The core lies in the configuration of the post_processing section:

    • module: Fixed to tricys.postprocess.baseline_analysis.
    • function: Fixed to baseline_analysis.
    • params:
      • detailed_var (string, optional):
        • Description: Specifies a variable in the model for which a detailed time-evolution plot will be generated in the report.
      • glossary_path (string, optional):
        • Description: Points to a "glossary" CSV file path. Providing this file greatly enhances the readability of the report by mapping abbreviated variable names (e.g., sds.I[1]) to understandable names and descriptions.

    4. Glossary File Format¶

    The glossary is a standard CSV file, and the recommended encoding format is UTF-8 with BOM, used to provide rich metadata for the report. Providing this file can greatly enhance the report's readability by mapping abbreviated variable names (e.g., sds.I[1]) to understandable names, terms, and detailed explanations.

    Its column headers should include:

    • 模型参数 (Model Parameter): (Required) The full variable name in the model, e.g., sds.I[1]. This is the key field used by the program to match data.
    • 英文术语 (English Term): (Optional) The standard English term for the variable, e.g., SDS Tritium Inventory.
    • 中文翻译 (Chinese Translation): (Optional) The Chinese name for the variable, e.g., 储存与输送系统. This name will be prioritized for display in the report.
    • 别名 (Alias): (Optional) Other common names or abbreviations for the variable, e.g., Storage Bed Inventory.
    • 说明 (Explanation): (Optional) A detailed explanation of the variable, its role in the system, or its physical meaning.

    Example (example_glossary.csv): 

    模型参数 (Model Parameter),英文术语 (English Term),中文翻译 (Chinese Translation),别名 (Alias),说明 (Explanation)
sds.I[1],SDS Tritium Inventory,储存与输送系统,Storage Bed Inventory / Reserve Inventory,"Tritium inventory in the Storage and Delivery System (SDS) sub-component."
div.I[1],DIV Tritium Inventory,偏滤器,Divertor Retention / Trapped Inventory,"Tritium inventory in the Divertor (DIV) sub-component."
blanket.TBR,Blanket Tritium Breeding Ratio,增殖包层 氚增殖比,TBR_actual / Breeding Performance,"The Tritium Breeding Ratio (TBR) of the blanket, key to fuel self-sufficiency (must be > 1)."
...

    5. Analysis Report Output¶

    After a successful run, tricys-ana will generate an interactive report named baseline_analysis_report.html in the results directory, which typically includes:

    1. Key Metrics Summary Table:
      • Displays key performance indicators (KPIs) such as the tritium inventory in various subsystems and the total inventory at the end of the simulation.
    2. Tritium Inventory Distribution Bar Chart:
      • Presents the tritium inventory of each subsystem in a bar chart for easy comparison.
    3. Detailed Variable Evolution Plot:
      • Shows the detailed curve of the variable specified by the detailed_var parameter over time.

    This report provides a comprehensive and intuitive view for evaluating and presenting the system performance under baseline conditions.
