    后处理模块¶

    tricys 不仅能运行仿真，还提供了一个强大的后处理（Post-Processing）框架，允许您在仿真任务完成后自动对结果进行分析、生成报告或执行其他自定义操作。

    后处理功能通过在配置文件中添加 post_processing 字段来实现。

    配置文件示例¶

    以下示例展示了如何在参数扫描完成后，自动执行两个分析任务：一个是“上升/下降时间分析”，另一个是“静态阈值报警检查”。

    {
    "paths": {
        ...
    },
    "simulation": {
        ...
    },
    "simulation_parameters": {
        "blanket.TBR": "linspace:1:1.5:10",
        "tep_fep.to_SDS_Fraction[1]":"log:0.1:0.99:5"
    },
    "post_processing": [
        {
            "module": "tricys.postprocess.rise_analysis",
            "function": "analyze_rise_dip",
            "params": {
                "output_filename": "rise_report.json"
            }
        },
        {
            "module": "tricys.postprocess.static_alarm",
            "function": "check_thresholds",
            "params": {
                "output_filename": "alarm_report.json",
                "rules": [
                    {
                        "columns": ["sds.I[1]"],
                        "min": 0.0
                    }
                ]
            }
        }
    ]
}

    配置项详解¶

    post_processing¶

    • 描述: 这是一个列表（Array），其中每个元素代表一个独立的后处理步骤。这些步骤会按照它们在列表中的顺序依次执行。
    • 执行时机: 所有仿真任务（例如参数扫描中的每一次运行）全部完成后，tricys 会将汇总的结果（sweep_results.csv）加载到一个 Pandas DataFrame 中，并将其作为第一个参数传递给您指定的后处理函数。

    后处理步骤对象¶

    列表中的每个对象都定义了一个要执行的 Python 函数，其结构如下：

    • module (字符串, 必填):

    • 描述: 要调用的 Python 函数所在的模块的完整路径。例如 tricys.postprocess.rise_analysis。

    • function (字符串, 必填):

    • 描述: 要在该模块中调用的函数名。例如 analyze_rise_dip。

    • params (字典, 选填):

    • 描述: 一个包含要传递给目标函数的关键字参数（keyword arguments）的字典。
    • 示例: 在 static_alarm 的例子中，params 提供了 output_filename 和 rules 两个参数。当 tricys 调用 check_thresholds 函数时，它会像这样执行： 
      # 伪代码
check_thresholds(
    results_df,  # 这是 tricys 自动传入的仿真结果
    output_filename="alarm_report.json",
    rules=[...]
)

    内置后处理模块¶

    tricys 自带了一些常用的后处理模块，位于 tricys/postprocess 目录下：

    • rise_analysis: 用于分析信号的上升时间、下降时间、峰值等动态特性。
    • static_alarm: 用于检查结果是否超出了预设的静态阈值（上限或下限）。
    • baseline_analysis: 用于执行基准工况分析。

    创建您自己的后处理模块¶

    后处理框架最大的优势在于其扩展性。您可以轻松编写自己的 Python 脚本来执行任何想要的分析。

    编写自定义函数的规则: 1. 您的函数第一个参数必须用于接收 Pandas DataFrame 格式的仿真结果。 2. 函数的其余参数可以通过 params 字段在 JSON 文件中自由定义。 3. 将您的 Python 脚本放在一个 tricys 可以导入的位置（例如，项目根目录下的一个新文件夹内，并确保有 __init__.py 文件）。

    通过这种方式，您可以将任意复杂的数据分析流程无缝集成到 tricys 的自动化工作流中。
