    快速开始¶

    本指南将引导您完成 tricys 的安装过程，并运行一个基本的命令行仿真。

    1. 环境要求¶

    在开始之前，请确保您的系统满足以下要求：

    • Python: 版本 3.8 或更高。
    • Git: 用于克隆项目仓库。
    • OpenModelica: 需要安装 OpenModelica，并确保其命令行工具 (omc.exe) 已添加到系统的 PATH 环境变量中。

    提示

    在 Windows 上安装 Python 时，请务必勾选 "Add Python to PATH" 选项，以确保 python 和 pip 命令在终端中可用。

    2. 安装步骤¶

    a. 克隆项目仓库¶

    打开您的终端（如 PowerShell 或 Cmd），使用 git 克隆 tricys 的源代码。

    git clone https://github.com/asipp-neutronics/tricys.git
cd tricys

    b. 创建并激活虚拟环境¶

    在项目根目录下，创建一个独立的 Python 虚拟环境，以隔离项目依赖。

    # 创建虚拟环境
py -m venv venv

# 激活虚拟环境
.\venv\Scripts\activate

    激活后，您会看到终端提示符前出现 (venv) 字样。

    c. 安装项目依赖¶

    使用 pip 安装 tricys 及其所有开发依赖项。-e 参数表示以“可编辑”模式安装，这意味着您对源代码的任何修改都会立即生效。

    pip install -e ".[dev]"

    3. 运行一个示例仿真¶

    tricys 附带了一个预先配置好的示例，可以帮助您快速验证安装是否成功。

    a. 准备配置文件¶

    项目中的 example/basic/1_basic_configuration/basic_configuration.json 是一个基础的仿真配置文件。您可以将其复制到项目根目录并重命名为 config.json，这是程序默认会查找的配置文件名。

    # 在 Windows 上使用 copy
copy "example\basic\1_basic_configuration\basic_configuration.json" "config.json"

    b. 运行仿真¶

    在激活了虚拟环境的终端中，执行以下命令来启动仿真：

    tricys

    由于存在默认的 config.json，tricys 会自动加载它并开始运行。您也可以通过 -c 参数显式指定配置文件：

    tricys -c config.json

    c. 查看结果¶

    仿真完成后，结果将保存在 results 目录下，并带有一个以运行时间命名的时间戳子目录，例如 results/20231116_143000/。

    在该目录中，您会找到： - simulation_result.csv: 包含所有输出变量随时间变化的数据。 - tricys.log: 本次运行的详细日志文件。 - config.json: 本次运行所使用的完整配置的备份。

    4. 运行图形用户界面 (GUI)¶

    如果您更喜欢图形化操作，可以启动 tricys 的 GUI。

    tricys gui

    GUI 提供了一个交互式界面，用于加载模型、设置参数、定义扫描范围和启动仿真。

    恭喜！您已经成功安装并运行了 tricys。接下来，您可以探索更高级的功能，例如参数扫描或协同仿真。
