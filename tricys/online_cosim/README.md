# 聚变堆系统模型联合仿真建模规范
**—— 面向 Tricys FMI/SSP 自动化构建与接管**

为了确保 Modelica 聚变堆系统模型能够无缝接入 Tricys `online_cosim`（基于 OMSimulator 的联合仿真总线），我们设计了一套高度自动化的 FMU/SSP 构建与封装流水线。

开发人员**无需**关注繁琐的底层通信细节与约束，**只需遵守一条最小化核心原则，即可保证模型完美兼容所有的自动化封装与高级 Python 代理控制。**

---

## 1. 核心自动化机制：从 Modelica 到 SSP 的无缝转换

Tricys 框架目前已实现对 Modelica 模型的全自动组网解析与控制权剥离，其底层工作流如下：

* **自动化编译与提取 (Auto-build & Extract)**：框架利用 OMC 引擎自动分析 `Cycle` 等顶层模型。它会将各个独立的子系统编译为标准的 FMU (Functional Mock-up Unit)，同时提取组件之间的网络图。
* **原生 SSP 封装与装配 (SSP Encapsulation)**：通过 OMSimulator，将所有生成的 FMU 和连接拓扑打包组装成标准的 `.ssp` 文件。这使得复杂的聚变物理模型成为一个标准化的 FMI 联合仿真黑盒。
* **动态切除与 Python 接管 (Dynamic Stripping & Proxy)**：当在 Python 侧配置了如 `AbstractTrackProcessor` 等高级拦截器时，Tricys 会在初始化 SSP 前，**利用 OMSimulator 底层 API 动态地将目标 FMU（如 `I_ISS`）从内存拓扑中安全删除 (Delete)**，以防止被接管的组件在后台继续运行并吞噬质量（形成“质量黑洞”）。
* **悬空端口无缝覆写 (Dangling Port Injection)**：被删除的组件留下的原输出网络连接将自动转化为下游节点的悬空引脚（如 `SDS.from_i_iss`），Python 拦截器正是通过覆写这些悬空的物理边界，实现了完全无痛的数据注入和质量守恒补偿。

## 2. 建模最小原则：顶层拓扑“零方程”原则 (Zero-Equation Top-Level Topology)

为了让上述自动化机制（特别是“动态切除”与悬空覆写）能够完美运作，**Modelica 建模只需要遵守唯一的黄金准则：顶层系统装配模型必须退化为纯粹的拓扑网络（Topological Netlist）。**

只要遵守此原则，系统模型将自然满足所有关于接口、观测及封装的苛刻要求：

* **完全依赖显式端口连接 (`connect`)**：顶层模型的 `equation` 区域只允许出现 `connect(A.port_out, B.port_in)`。这强制模型实现了显式的因果端口解耦，使得 Tricys 在自动编译时能准确抓取数据流向，并完美映射至 SSP 标准结构。
* **杜绝隐式状态依赖（禁止跨组件点操作符）**：严禁在顶层使用如 `A.x = B.y` 直接赋值或读取。由于 Python 代理可能会在运行时物理切除某个原生组件，任何隐式的点操作符查询都会在实例化时直接引发由于“目标组件不存在”而导致的运行崩溃。
* **控制逻辑天然下沉或剥离**：不允许在顶层编写微分方程或 PID 控制逻辑（如 `der(x)` 或复杂的 `if/when`）。所有系统级控制必须要么下沉至独立的 FMU 组件内，要么外置剥离交由 Tricys 的 `Auditor` 和 `Python Wrappers` 统一管理。
* **免除冗余探测代码**：凡是封装好的独立 FMU，其内部状态变量（如 `inventory`, `leak`）在编译后都会静态存在于对应的 `modelDescription.xml` 中。Tricys 的审计员 (`Auditor`) 会通过后台正则表达式直接深度扫描提取，开发者完全无需为了外部监控而在 Modelica 层面额外连线或引出多余的物理观测端口。

### 总结
**“零方程”原则不是一种限制，而是为了实现模型最高级别的“高内聚、低耦合”。** 

执行本规范的核心目的，是将 Modelica 语言的使用范式从“随意挥洒的方程集”约束为“严谨的系统工程（Systems Engineering）架构”。在 Tricys 的体系下，**Modelica 专职负责局部物理域的动态建模及绘制连接拓扑蓝图；而全局的质量/能量守恒审计、复杂的跨域业务调度以及高阶的动态误差补偿控制，则统统交由更高维度的 Python 框架进行降维管理。**

只要聚变系统在顶层表现为基于 `connect` 连接的纯净图纸，无论未来如何对单个物理节点进行 AI 替代、降维代理或全局误差补偿，Tricys 框架都能游刃有余地完成自动化的外科手术式切除与重新缝合。
