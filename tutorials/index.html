    教程（Tutorials）¶

    本章节通过一系列示例，指导您从基础到高级使用 tricys 的各项功能。所有示例均位于项目根目录的 example/ 文件夹下。

    OpenModelica模型¶

    tricys 的核心是基于 OpenModelica 进行物理建模和仿真。所有的模型都由 .mo 文件定义。在 tricys 中，通常使用一个 package.mo 文件来组织一个完整的模型库。

    您可以在 example/example_model/ 目录中找到一个完整的示例模型，它定义了聚变燃料循环中的各个子系统（如增殖包层 Blanket、同位素分离系统 I_ISS 等）及其相互作用。

    中英文专业术语表¶

    下表整理了模型中的关键参数和分析指标，摘自 example/example_glossary/example_glossary.csv。

    模型参数 (Model Parameter) 英文术语 (English Term) 中文翻译 (Chinese Translation) 说明 (Explanation)
    pulseSource.amplitude Pulse Amplitude 脉冲幅度 系统驱动：设定氚的消耗速率 (g/h)。
    pulseSource.period Pulse Period 运行周期 模拟“运行-停机”的总循环时长。
    pulseSource.width Availability Factor 占空比 反应堆运行时间占总周期的百分比（%）。
    blanket.TBR Blanket Tritium Breeding Ratio 增殖包层 氚增殖比 燃料自持的关键 (必须 > 1)。
    blanket.T Blanket Mean Residence Time 增殖包层 平均滞留时间 增殖包层中氚的平均滞留时间。
    i_iss.T I_ISS Mean Residence Time 同位素分离I路 平均滞留时间 工艺延迟，I_ISS（低温精馏）是主要的库存持有者。
    plasma.fb Plasma Burn Fraction 等离子体 燃烧分数 注入燃料中实际聚变的比例。
    plasma.nf Plasma Fueling Efficiency 等离子体 燃料效率 注入燃料中实际进入芯部的比例。
    分析指标
    Startup_Inventory Start up Inventory 启动库存 反应堆首次启动前必须提供的初始氚库存量。
    Required_TBR Required Tritium Breeding Ratio 所需氚增殖比 为实现氚自持所必须达到的最低TBR。
    Doubling_Time Doubling Time (库存)倍增时间 一个反应堆产生足够启动另一个新反应堆所需氚库存的时间。
    Self_Sufficiency_Time Self Sufficiency Time (氚)自持时间 氚库存由下降趋势转变为上升趋势的拐点时间。

    TRICYS BASIC¶

    这部分介绍 tricys 的基础用法。

    基础配置 (Basic Configuration)¶

    这是最简单的用法：对一个给定的模型和参数，运行一次仿真。

    • 示例目录: example/basic/1_basic_configuration/
    • 运行命令: 
      # 确保您在 tricys 的项目根目录下
tricys -c example/basic/1_basic_configuration/basic_configuration.json
    • 配置文件: basic_configuration.json 
      {
    "paths": {
        "package_path": "../../example_model_single/example_model.mo"
    },
    "simulation": {
        "model_name": "example_model.Cycle",
        "variableFilter": "time|sds.I[1]",
        "stop_time": 2000.0,
        "step_size": 0.5
    }
}
      说明:
    • package_path: 指向模型文件。
    • model_name: 指定要仿真的顶层模型。
    • variableFilter: 指定需要输出的变量，用|分隔。
    • stop_time, step_size: 定义仿真时长和步长。

    参数扫描 (Parameter Sweep)¶

    研究单个或多个参数变化对仿真结果的影响。

    • 示例目录: example/basic/2_parameter_sweep/
    • 运行命令: 
      tricys -c example/basic/2_parameter_sweep/parameter_sweep.json
    • 配置文件: parameter_sweep.json 
      {
    // ... simulation部分与基础配置相同 ...
    "simulation_parameters": {
        "tep_fep.to_SDS_Fraction[1]":[0.1,0.15,0.2,0.3,0.4,0.6,0.8]
    }
}
      说明:
    • simulation_parameters: 在这里定义要扫描的参数。
    • "tep_fep.to_SDS_Fraction[1]": 要扫描的模型参数名。
    • [0.1, 0.15, ...] : 一个包含所有扫描值的列表。tricys会为列表中的每个值运行一次仿真。

    并发运行 (Concurrent operation)¶

    当参数扫描任务很多时，可以开启并发运行以利用多核CPU加速计算。

    • 示例目录: example/basic/3_concurrent_operation/
    • 运行命令: 
      tricys -c example/basic/3_concurrent_operation/concurrent_operation.json
    • 配置文件: concurrent_operation.json 
      {
    "simulation": {
        // ...
        "concurrent": true
    },
    "simulation_parameters": {
        "blanket.TBR": "linspace:1:1.5:10"
    }
}
      说明:
    • "concurrent": true: 开启并发模式。
    • "linspace:1:1.5:10": 一种高级参数定义格式，表示在1到1.5之间生成10个等间距的数值进行扫描。

    后处理模块 (Post-processing module)¶

    在仿真结束后，自动调用Python脚本对结果进行分析、绘图或生成报告。

    • 示例目录: example/basic/4_post_processing_module/
    • 运行命令: 
      tricys -c example/basic/4_post_processing_module/post_processing_module.json
    • 配置文件: post_processing_module.json 
      {
    // ... simulation_parameters 定义了二维扫描 ...
    "post_processing": [
        {
            "module": "tricys.postprocess.rise_analysis",
            "function": "analyze_rise_dip",
            "params": { "output_filename": "rise_report.json" }
        },
        {
            "module": "tricys.postprocess.static_alarm",
            "function": "check_thresholds",
            "params": { /* ... */ }
        }
    ]
}
      说明:
    • post_processing: 一个列表，定义了要执行的后处理任务。
    • module, function: 指定要调用的Python模块和函数。
    • params: 传递给该函数的参数。

    协同仿真模块 (Co-simulation module)¶

    将 Modelica 模型与外部程序（如Aspen Plus、Python脚本）连接起来，实现混合仿真。

    • 示例目录: example/basic/5_co_simulation_module/
    • 运行命令: 
      tricys -c example/basic/5_co_simulation_module/co_simulation_module.json
    • 配置文件: co_simulation_module.json 
      {
    // ...
    "co_simulation": [
        {
            "submodel_name": "example_model.DIV",
            "instance_name": "div",
            "handler_module": "tricys.handlers.div_handler",
            "handler_function": "run_div_simulation",
            "params": { /* ... */ }
        },
        {
            "submodel_name": "example_model.I_ISS",
            "instance_name": "i_iss",
            "handler_module": "tricys.handlers.i_iss_handler",
            "handler_function": "run_aspen_simulation",
            "params": {
                "bkp_path": "../../example_aspenbkp/T2-Threetowers4.bkp",
                // ...
            }
        }
    ]
}
      说明:
    • co_simulation: 定义协同仿真任务列表。
    • submodel_name: 要被外部程序替代的Modelica子模型。
    • handler_module, handler_function: 指定用于替代该子模型的Python处理函数。
    • params: 传递给处理函数的参数，例如Aspen Plus模型文件路径 bkp_path。

    TRICYS ANALYSIS¶

    这部分介绍 tricys 内置的高级分析功能，通过 sensitivity_analysis 配置块实现。

    基准工况分析 (Baseline condition analysis)¶

    对单次仿真结果进行详细的后处理分析，并生成格式化的报告。

    • 示例目录: example/analysis/1_baseline_condition_analysis/
    • 运行命令: 
      tricys-ana -c example/analysis/1_baseline_condition_analysis/baseline_condition_analysis.json
    • 配置文件: baseline_condition_analysis.json 
      {
    // ... simulation ...
    "post_processing": [
        {
           "module": "tricys.postprocess.baseline_analysis",
           "function": "baseline_analysis",
           "params": {
                "detailed_var": "sds.I[1]",
                "glossary_path": "../../example_glossary/example_glossary.csv"
            }
        }
    ]
}
      说明: 此功能通过一个专用的后处理脚本 baseline_analysis 实现，它会加载仿真结果和术语表，生成一份图文并茂的基准工况分析报告。

    单参数敏感性分析 (Single-parameter sensitivity analysis)¶

    系统地研究单个独立参数的变化对多个关键性能指标（如启动库存、自持时间）的影响。

    • 示例目录: example/analysis/2_single_parameter_sensitivity_analysis/
    • 运行命令: 
      tricys-ana -c example/analysis/2_single_parameter_sensitivity_analysis/single_parameter_sensitivity_analysis.json
    • 配置文件: single_parameter_sensitivity_analysis.json 
      {
    "sensitivity_analysis": {
        "enabled": true,
        "analysis_cases": [
            {
                "name": "TBR_Analysis",
                "independent_variable": "blanket.TBR",
                "independent_variable_sampling": [1.05,1.1,1.15,1.2],
                "dependent_variables": [
                    "Startup_Inventory",
                    "Self_Sufficiency_Time",
                    "Doubling_Time"
                ],
                // ...
            }
        ],
        "metrics_definition": {
            "Startup_Inventory": {
                "source_column": "sds.I[1]",
                "method": "calculate_startup_inventory"
            },
            // ...
        }
    }
}
      说明:
    • sensitivity_analysis: 敏感性分析的总开关和配置。
    • analysis_cases: 定义一个或多个分析案例。
    • independent_variable: 要改变的自变量。
    • dependent_variables: 要观察的因变量（性能指标）。
    • metrics_definition: 定义如何从原始仿真结果（如sds.I[1]的时间序列）计算出性能指标（如Startup_Inventory）。

    多参数敏感性分析 (Multi-parameter sensitivity analysis)¶

    同时研究多个参数变化对性能指标的影响，常用于生成二维图谱。

    • 示例目录: example/analysis/3_multi_parameter_sensitivity_analysis/
    • 运行命令: 
      tricys-ana -c example/analysis/3_multi_parameter_sensitivity_analysis/multi_parameter_sensitivity_analysis.json
    • 配置文件: multi_parameter_sensitivity_analysis.json 
      {
    "sensitivity_analysis": {
        // ...
        "analysis_cases": [
            {
                "name": "DIR_PLASMA_Analysis",
                "independent_variable": "tep_fep.to_SDS_Fraction[1]",
                "independent_variable_sampling": [0.1,0.3,0.6,0.8],
                "dependent_variables": ["Startup_Inventory", "Required_TBR"],
                "simulation_parameters": {
                    "plasma.fb": [0.02,0.04,0.08,0.09,0.1]
                }
            }
        ]
    }
}
      说明: 与单参数分析类似，但在 analysis_cases 中额外增加了一个 simulation_parameters 块，用于定义第二个扫描维度。

    SOBOL全局敏感性分析 (SOBOL global sensitivity analysis)¶

    使用 Sobol' 方法，在多维参数空间中进行采样和分析，量化每个参数对输出方差的贡献，从而识别出最重要的影响参数。

    • 示例目录: example/analysis/4_sobol_global_sensitivity_analysis/
    • 运行命令: 
      tricys-ana -c example/analysis/4_sobol_global_sensitivity_analysis/sobol_global_sensitivity_analysis.json
    • 配置文件: sobol_global_sensitivity_analysis.json 
      {
    "sensitivity_analysis": {
        "analysis_cases": [
            {
                "name": "SALIB_SOBOL_Analysis",
                "independent_variable":["pulseSource.width","plasma.nf", ...],
                "independent_variable_sampling":{
                      "pulseSource.width": { "bounds": [50,90] },
                      // ...
                },
                "analyzer": {
                    "method": "sobol",
                    "sample_N": 256
                }
            }
        ]
    }
}
      说明:
    • independent_variable: 变为一个包含多个参数的列表。
    • independent_variable_sampling: 定义每个参数的采样边界 bounds。
    • analyzer: 指定使用 sobol 方法，并设置采样数 sample_N。

    LATIN不确定性分析 (LATIN Uncertainty Analysis)¶

    使用拉丁超立方采样 (Latin Hypercube Sampling) 方法，在多维参数空间中高效地采样，用于评估模型输出的不确定性范围。

    • 示例目录: example/analysis/5_latin_uncertainty_analysis/
    • 运行命令: 
      tricys-ana -c example/analysis/5_latin_uncertainty_analysis/latin_uncertainty_analysis.json
    • 配置文件: latin_uncertainty_analysis.json 
      {
    "sensitivity_analysis": {
        "analysis_cases": [
            {
                // ... 与Sobol配置类似 ...
                "analyzer": {
                    "method": "latin",
                    "sample_N": 256
                }
            }
        ]
    }
}
      说明: 配置与Sobol分析几乎完全相同，只是将 analyzer.method 更改为 latin。
